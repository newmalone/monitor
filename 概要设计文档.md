# 设备状态监控系统概要设计文档

## 1. 总体架构
系统采用前后端分离架构：
- 前端：React + TypeScript + Ant Design + Vite
- 后端：Node.js + Express
- 数据存储：JSON 文件快照存储
- 地图服务：高德地图 JS API

## 2. 模块设计

### 2.1 前端模块
- `App.tsx`：整体布局、菜单、路由
- `pages/MonitorPage.tsx`：监控主页面
- `pages/ComparePage.tsx`：双日期比对页面
- `pages/HistoryPage.tsx`：历史数据查询页面
- `components/FileUploader.tsx`：文件上传组件
- `components/StatsCard.tsx`：状态概览卡片
- `components/ChartPanel.tsx`：统计图表与报表展示
- `components/DeviceMap.tsx`：地图组件
- `components/DeviceTable.tsx`：设备明细表格

### 2.2 后端模块
- `server/index.js`：接口服务、上传处理、Excel 解析
- `server/db.js`：快照与设备数据 JSON 文件存取

### 2.3 服务模块
- `src/services/api.ts`：前端接口调用
- `src/services/stats.ts`：统计计算
- `src/services/comparison.ts`：双日期对比计算

## 3. 数据流设计
1. 用户上传 Excel 文件
2. 前端仅上传文件与日期
3. 后端解析 Excel 首个工作表
4. 后端转换为设备数组并保存为快照
5. 前端接收解析结果并刷新页面

## 4. 存储设计
- `server/data/snapshots.json`：快照索引
- `server/data/devices_YYYY-MM-DD.json`：某日设备快照明细

### 快照索引结构
- date
- sourceFile
- importedAt
- totalCount
- enabledCount

### 设备结构
包含设备编号、产品名称、厂商、类型、区域、经纬度、状态、启用状态等字段。

## 5. 接口设计

### POST /api/upload
- 入参：`file`、`date`
- 功能：上传并解析 Excel，保存快照
- 返回：日期、总数、已启用数、设备列表

### GET /api/snapshots
- 功能：获取全部历史快照

### GET /api/snapshots/latest
- 功能：获取最新快照

### GET /api/snapshots/:date
- 功能：获取某日完整快照

### GET /api/devices/:date
- 功能：获取某日设备列表

### DELETE /api/snapshots/:date
- 功能：删除某日快照

## 6. 异常处理设计
- 非 xlsx 文件拒绝上传
- 文件过大拒绝上传
- 无工作表文件返回错误
- 上传失败返回统一 JSON 错误消息
- 前端对网络异常、后端 JSON 异常、文本异常分别提示

## 7. UI 设计原则
- 左侧菜单导航
- 顶部面包屑和标题区
- 卡片化展示统计数据
- 深色侧边栏 + 浅色内容区
- 风格参考 vue-pure-admin

## 8. 当前版本技术结论
当前采用 JSON 文件存储替代 SQLite，降低部署复杂度，适合本地及局域网共享场景。
